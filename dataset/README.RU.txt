Резюме
=======

Этот набор данных (ml-latest-small) описывает 5-звездочный рейтинг и активность с тегами свободного текста от [MovieLens](http://movielens.org ), сервис рекомендаций фильмов. Он содержит 100836 оценок и 3683 приложения с тегами для 9742 фильмов. Эти данные были созданы 610 пользователями в период с 29 марта 1996 года по 24 сентября 2018 года. Этот набор данных был сгенерирован 26 сентября 2018 года.

Пользователи были выбраны случайным образом для включения. Все выбранные пользователи оценили не менее 20 фильмов. Демографическая информация не включена. Каждый пользователь представлен идентификатором, и никакая другая информация не предоставляется.

Данные содержатся в файлах `links.csv`, `movies.csv`, `ratings.csv` и `tags.csv`. Более подробная информация о содержимом и использовании всех этих файлов приведена ниже.

Это набор данных для разработки. Как таковой, он может меняться с течением времени и не является подходящим набором данных для совместного использования результатов исследований. Смотрите доступные * эталонные * наборы данных, если это ваше намерение.

Этот и другие наборы данных GroupLens общедоступны для загрузки по адресу <http://grouplens.org/datasets />.

Содержание и использование файлов
========================

Форматирование и кодирование
-----------------------

Файлы набора данных записываются в виде [значений, разделенных запятыми](http://en.wikipedia.org/wiki/Comma-separated_values ) файлы с одной строкой заголовка. Столбцы, содержащие запятые (`,`), экранируются с помощью двойных кавычек (`"`). Эти файлы закодированы как UTF-8. Если символы с ударением в названиях фильмов или значениях тегов (например, Misérables, Les (1995)) отображаются неправильно, убедитесь, что любая программа, считывающая данные, такая как текстовый редактор, терминал или скрипт, настроена для UTF-8.


Идентификаторы пользователей
--------

Пользователи MovieLens были выбраны случайным образом для включения. Их идентификаторы были анонимизированы. Идентификаторы пользователей совпадают между `ratings.csv` и `tags.csv` (т.Е. Один и тот же идентификатор относится к одному и тому же пользователю в двух файлах).


Идентификаторы фильмов
---------

В набор данных включаются только фильмы, имеющие хотя бы один рейтинг или тег. Эти идентификаторы фильмов соответствуют тем, которые используются на веб-сайте MovieLens (например, идентификатор `1` соответствует URL-адресу <https://movielens.org/movies/1 >). Идентификаторы фильмов совпадают между `ratings.csv`, `tags.csv`, `movies.csv` и `links.csv` (т.Е. Один и тот же идентификатор относится к одному и тому же фильму в этих четырех файлах данных).


Структура файла данных рейтингов (ratings.csv)
-----------------------------------------

Все рейтинги содержатся в файле `ratings.csv`. Каждая строка этого файла после строки заголовка представляет одну оценку одного фильма одним пользователем и имеет следующий формат:

    Идентификатор пользователя, идентификатор фильма, рейтинг, временная метка

Строки в этом файле упорядочены сначала по идентификатору пользователя, затем, внутри пользователя, по идентификатору фильма.

Оценки составляются по 5-звездочной шкале с шагом в ползвезды (0,5 звезды - 5,0 звезды).

Временные метки представляют секунды с полуночи по Всемирному координированному времени (UTC) 1 января 1970 года.


Структура файла данных тегов (tags.csv)
-----------------------------------

Все теги содержатся в файле `tags.csv`. Каждая строка этого файла после строки заголовка представляет один тег, примененный к одному фильму одним пользователем, и имеет следующий формат:

    Идентификатор пользователя, идентификатор фильма, тег, временная метка

Строки в этом файле упорядочены сначала по идентификатору пользователя, затем, внутри пользователя, по идентификатору фильма.

Теги - это созданные пользователем метаданные о фильмах. Каждый тег обычно представляет собой одно слово или короткую фразу. Смысл, ценность и назначение конкретного тега определяются каждым пользователем.

Временные метки представляют секунды с полуночи по Всемирному координированному времени (UTC) 1 января 1970 года.


Структура файла данных фильмов (movies.csv)
---------------------------------------

Информация о фильме содержится в файле `movies.csv". Каждая строка этого файла после строки заголовка представляет один фильм и имеет следующий формат:

    Идентификатор фильма, название, жанры

Названия фильмов вводятся вручную или импортируются из <https://www.themoviedb.org />, и укажите год выпуска в скобках. В этих названиях могут существовать ошибки и несоответствия.

Жанры представляют собой список, разделенный каналами, и выбираются из следующих:

* Действие
* Приключение
* Анимация
* Детская
* Комедия
* Преступление
* Документальный фильм
* Драма
* Фантазия
* Фильм-нуар
* Ужас
* Музыкальный
* Тайна
* Романтика
* Научно-фантастический
* Триллер
* Война
* Западный
* (жанры не указаны)


Структура файла данных ссылок (links.csv)
---------------------------------------

Идентификаторы, которые можно использовать для ссылки на другие источники данных о фильмах, содержатся в файле `links.csv`. Каждая строка этого файла после строки заголовка представляет один фильм и имеет следующий формат:

    Идентификатор фильма,imdbId,tmdbId

MovieID - это идентификатор фильмов, используемых <https://movielens.org >. Например, в фильме "История игрушек" есть ссылка <https://movielens.org/movies/1 >.

imdbId - это идентификатор фильмов, используемых <http://www.imdb.com >. Например, в фильме "История игрушек" есть ссылка <http://www.imdb.com/title/tt0114709 />.

tmdbId - это идентификатор фильмов, используемых <https://www.themoviedb.org >. Например, в фильме "История игрушек" есть ссылка <https://www.themoviedb.org/movie/862 >.

Использование ресурсов, перечисленных выше, регулируется условиями каждого поставщика.

----------------

Мы больше не объединяем ни одну из этих функций с набором данных, поскольку большинство современных наборов инструментов предоставляют это как встроенную функцию. Если вы хотите узнать о стандартных подходах к перекрестным вычислениям в контексте оценки рекомендательных систем, см. [LensKit](http://lenskit.org ) для инструментов, документации и примеров кода с открытым исходным кодом.
